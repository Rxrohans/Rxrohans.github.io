Low Back Pain Tracking App

A Digital Diary and Trend Analysis Tool

Problem Statement

Low back pain is a chronic issue affecting millions. Tracking it subjectively is difficult, and correlating it with factors like sleep and activity is usually neglected by patients and clinicians.

Objectives

  • Track pain levels across different times of the day (morning, afternoon, evening).
  • Include influencing parameters like sleep quality, medication intake, and physical activity.
  • Enable trend analysis using data visualization tools.

Solution & Outcomes

Successfully developed and deployed an android mobile application for tracking low back pain. • The app was able to collect daily qualitative data from patients using a predefined set of questions. • Secure data storage with Google sheets integration, and SQLite database for offline data handling. • A pilot testing of the app was done to ensure the functionality and fix bugs.

Skills Used

  • Python (pandas, matplotlib), Kotlin
  • Android Studio
  • UI/UX wireframing and prototyping
